## 1.文件的类型

### 1.1 目录相关

#### 1.1.1  目录的介绍

```

   查看跟目录：ls /
    bin :存放二进制的命令，可执行文件
    boot：启动有关的
    dev: 设备有关的 光驱
    etc:配置文件
    home：用户的家目录
    lib:库文件
    proc:进程内存状态映射
    root：管理员的家目录
    sbin:命令
    sys:系统
    tmp:临时文件
    var:日志
```

#### 1.1.2 目录的操作

```python
1. 目录的创建：

   # mkdir /oldboy
   # mkdir -p /a/b/c  同时迭代的创建多个
   # mkdir a{1..10} 同时创建10个文件 

2.目录的查询：
    
    # ls -ld /oldboy  --> 详细的信息
    # ls -ltr: 按照时间的倒叙排列显示，最新的文件在最后面
    # ls -al : 查看所有文件 包括隐藏文件
    # ls -ld dir* 模糊查询
    # tree  /a  可以看到跟目录下的所有的子目录
    # tree -L 1 / 只看跟目录的1层
   
 
3. 目录的删除：

 # rm -rf /oldboy  (r: 目录 f:强制)  可以递归的删除
 # rm -rf /dir* 模糊删除
 
4. 修改(剪切 复制 重命名)
 
  #1. mv /root/oldboy  /temp  （剪切+复制）
  #2. mv /root/oldby /temp/oldboy.bak (剪切+复制+重命名)
 
5. 复制

  # cp -r /temp/oldboy  /  (将 oldboy的目录复制到根目录下，r:目录)
  # cp -a  /temp/oldboy  / （带着权限一起复制）
```

## 3.用户 组 权限

### 3.1 用户组的操作

```python
1. 组的操作

# 创建：groupadd -g 1001 dev 

# 看： cat /etc/group | tail -3 /etc/group

# 删除： groupdel dev

# 修改： groupmod -g 10086 dev

2. 用户的操作

# 增加： useradd oldboy | useradd -u 1001 -g dev（10086）

# 看：id oldboy

# 设置密码： passwd oldboy 

# 删除：userdel -r oldboy

# 修改：usermod -u 3000 oldboy

# 切换：su - oldboy 

# 退出： exit
```


## 7. Raid

```
功能特性：
    1.将多块磁盘合并成一块磁盘,提供更大的存储空间
	2.可以提供更高的IO能力
	3.数据物理层面的高安全

Raid的工作级别:
    raid0:条带化功能,性能极高,安全性和单盘一样
          至少两块盘,理论上盘越多性能越高

    raid1:镜像功能,性能没有明显提升,安全性高.	
          浪费一半空间

    raid10:镜像+条带化 ,至少四快盘,性能和安全兼顾
          浪费一般空间	

    raid5:带有校验功能的条带化	
         存储数据时,根据数据计算校验值,存储到第三块盘.
         写入性能较低
         读数据性能较高
         少三快盘,只允许一块盘损坏,浪费1/n的磁盘空间

         适合于读多写少的场景	
   
```
