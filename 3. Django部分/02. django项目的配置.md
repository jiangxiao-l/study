# django项目的配置

```
 --- APP的配置信息
		查看settings文件中的INSTALLED_APPS 是否配置了 'app01.apps.App01Config'
 
 --- 基于post传输数据的配置
      将MIDDLEWARE 这个列表中的 'django.middleware.csrf.CsrfViewMiddleware',删除
      
 --- 查看 template 
 		DIRS': [os.path.join(BASE_DIR, 'templates')]
 		
  --- 数据库的相关配置：
    DATABASES = {
        'default': {
            'ENGINE': 'django.db.backends.mysql',
            'PORT':3306,
            'HOST':'127.0.0.1',
            'USER':'root',
            'PASSWORD':'admin',
            'NAME': 'user_info',
        }
}

---静态文件的相关配置：
  // 这里的static 与在HTML文件中导入静态文件时的一致
  STATIC_URL = '/static/'
  
  STATICFILES_DIRS=[
    os.path.join(BASE_DIR,'static')
    // 这里的static 与手动创建的文件夹一致   
  ]


    -- 1.在死静态文件：<link rel="stylesheet" href="/static/css/mycss.css">
    
    --- 2.使用static标签函数
          {% load static %}   static 的返回值上传参的路径
	     
          Eg:		
          {% load static %}
          <link rel="stylesheet" href="{% static 'css/mycss.css' %}">

     ---3. Get_static_prefix 标签
             {% load static %}  
             {% get_static_prefix %}css/my.css    get_static_prefix返回的是：静态文件的地                                                       址，相当于 /static/
           Eg:
		      {% load static %}
              <link rel="stylesheet" href="{% get_static_prefix %}css/mycss.css">

 
```
