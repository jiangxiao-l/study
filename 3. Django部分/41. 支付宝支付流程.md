# 支付宝支付流程

```python
1. 正式环境:用营业执照,申请商户号,appid
2. 测试环境:沙箱环境:https://openhome.alipay.com/platform/appDaily.htm?tab=info
3. 支付宝提供接口:给商户使用,收钱
4. Java,php,C#的demo,没有python的demo
5. git有人封装了	
6. 需要安装模块:# pip3 install pycryptodome
7. 应用私钥---自己保存,一定不能丢
8. 应用公钥---给别人用
9. 支付宝公钥---支付宝用的
10.生成公钥私钥:https://docs.open.alipay.com/291/105971
11.把应用公钥配置在支付宝上:应用公钥,配置完成以后,支付宝自动生成一个支付宝公钥	
12.在程序中:配置应用私钥,支付宝公钥	
13 #如果支付成功,支付宝会回调,但是如果你的服务器挂掉了怎么办?
			-支付宝24小时以内不定时再给你发,你修改掉订单状态即可
14. 支付成功,支付宝会有一个get回调,一个post回调:修改订单状态
15.要是有别人的demo

16. 在demo文件夹中：
   1. alipay_public_2048.txt 文件存放的是 # 支付宝提供的公钥(是在生成公钥的时候自动生产的)
   2. app_private_2048.txt   文件存放的是 # 用户的私钥(是在生成公钥的时候也会生成)


17.# 用户单机支付按钮，重定向到支付宝的支付页面，用户扫码支付，一旦成功，支付宝会向咱们的程序，发送两个请求，一个是get请求，一个是post请求。你的地址一定是有公网的ip，是为了支付宝回调的时候能成功。同常情况下：get请求是展示一个页面，post请求通常是来修改订单的状态。

18.# 面试：可以封装一个类。傻瓜式的配置，就可以实现任意的方式推送信息。还可以实现规定几个人的推送方式。
```
